## 📚 第2块：文档摄取 + 向量化引擎

> **目标**：能把 Obsidian 里的 .md / .txt 书籍内容变成可检索的向量

### 任务清单

- [x] 2.1 实现文档加载器 —— 递归扫描 Obsidian 文件夹，读取 .md/.txt
- [x] 2.2 实现文本分块器 —— 按段落/固定长度切分文档
- [x] 2.3 实现嵌入模型接口 —— 支持配置多种嵌入模型
- [x] 2.4 集成向量数据库（ChromaDB 本地存储）
- [x] 2.5 实现 `ingest` 命令 —— 一键摄取指定文件夹所有文档
- [x] 2.6 测试：用示例文档验证检索效果

### 产出物

| 文件 | 说明 |
|------|------|
| `src/ingest/loader.py` | 文档加载器 |
| `src/ingest/chunker.py` | 文本分块器 |
| `src/ingest/embedder.py` | 嵌入模型接口 |
| `src/ingest/vector_store.py` | 向量数据库封装 |
| `src/ingest/pipeline.py` | 摄取管道（串联以上模块） |

## 🧠 第3块：查询推理引擎 + 打包分发

> **目标**：输入问题 → 检索相关段落 → LLM 推理 → 输出答案

### 任务清单

- [x] 3.1 实现 LLM 接口 —— 支持配置多种模型（Claude / OpenAI 等）
- [x] 3.2 实现检索器 —— 根据问题从向量库中召回最相关段落
- [x] 3.3 实现 RAG 推理管道 —— 拼接上下文 + 调用 LLM 生成回答
- [x] 3.4 实现 `ask` 命令 —— 交互式问答
- [x] 3.5 实现 `status` 命令 —— 查看已索引的文档统计
- [x] 3.6 完善打包分发脚本（`setup.py` / `pip install -e .`）
- [x] 3.7 端到端测试：摄取一本书 → 提问 → 得到有依据的回答

### 产出物
